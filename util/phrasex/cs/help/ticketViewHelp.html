Help : View Request
Respond vs. Edit
The "Respond" command allows you to respond to a request by entering a note. Both the "Respond" command and the "Edit" command allow you to enter a note. The difference between these commands is that the note entered from the "Respond" command will be sent to its submitters by email when the Email Sender is enabled. In addition, the note is always a public one and is visible to the submitter.
Divide command
Often, people submit a request with multiple questions. They may also ask a new question unrelated to a previous one. The Divide command allows you to create a new request based on the current one. That way, you can allocate one question per request. Note, the Divide command will bring up a new window. If you don't see it, check the Taskbar.
Co-submitter Email
A person who submitted a request may want all communications to be copied to other people. This works like the Cc: field of an email. You can enter multiple email addresses separated by commas.
Submitted To
The "Submitted To" field corresponds to the staff member who entered the request, received over the phone or by fax, for an End User.
