Build Build phrases list dialog window

Choose Tools|Build phrases list of all documents command to display the Build phrases list dialog window.

Specify file name of list of documents you want to include to the phrases list and set options that affect the search.

Then press Start button or press Enter key. Building of index files will be started.

If you want to cancel building, press Stop button. After index files created, press Cancel button

Created list of phrases is useful to translate set of documents by editing list of phrases and corresponding translation. Created list stored in the same folder where input listing of file (.lst) file resides and this file is looking like as following example:

Aedes (Ochlerotatus) mercurator
Aedes (Ochlerotatus) punctor
Aedes (Ochlerotatus) rempeli
Aedini
aino Viol.
Anasimyia
...

Note that all phrases are stored in alphabetical order (case insensitive or, if appropriate option is set, case sensitive).

Input (.lst) file contains numbered list of files to be included in the phrase list. First line indicates date and time when this listing was created. Each line below the first line consists of number (ordered form 1), tabulator symbol and relative or absolute path of the file. For example, this is short listing:

25.05.2004 21:12:28
1 0002.txn
2 0003.txn

In the dialog you can enter new file name of listing or select existing listing file. To do that, click ... button to browse folders. 

In case of listing file does not exists, you are prompted to create new ones. Press "Yes" to include all files recursively (in other words, include all subfolders in desired directory) or "No" to include files just in selected directory. 

Note that only files with extension of current document type filter of the Project window will be included in the new created listing.

If you would like include more files, you can use Find text dialog. Find text dialog have option to scan directories and create listing file. Also you can specify text to find in documents, so you may omit some documents from listing.

Other way is - create some listing files and then merge them in one big listing file. There are no matter of  number of file, also you can leave date and time stamp in the merged listing. Please note that just first line contains valid date and time stamp. 

Dialog box options

Option Meaning
Case sensitive This option is intended to include phrases case-sensitive. It is not recommended to checking up this option in any case.
Output unicode text (UTF-8) If you documents contains text in different character code pages, different from ANSI and default codepage of your Windows version, provide this option. Otherwise, all text in your documents will be included to the phrases list converted to ANSI character set (plus extended characters, visible in your version of Windows). For example, if your text includes Greek characters or mathematic formulas, mark up this check box.

Note: case sensitive option is ignored if this option is set.
